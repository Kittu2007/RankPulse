1. Executive Summary

RankPulse is a production-grade Social Media SEO Intelligence Platform designed to help creators, brands, agencies, and social media managers maximize organic reach across Instagram, LinkedIn, and X (Twitter). The platform provides AI-powered content analysis, real-time SEO scoring, keyword research, competitor intelligence, hashtag strategy, and an automated optimization engine — all powered by the latest 2025-2026 platform algorithms.



Unlike generic scheduling tools, RankPulse is purpose-built for social SEO with platform-specific intelligence baked into every feature. The system is not vibe-coded — it is engineered around verified algorithm signals, open-sourced ranking code (X), official platform documentation (Instagram/Meta, LinkedIn Engineering Blog), and continuous research from industry leaders.



🎯  Mission: Give every creator and brand the unfair SEO advantage on social — free from guesswork.

📊  Target Users: Content creators, SMM agencies, B2B marketers, e-commerce brands, consultants.

🏆  Competitive Benchmarks: Semrush Social, Sprout Social, Metricool, Hootsuite, Later, Taplio.

💡  Core Differentiator: Platform-specific algorithm-aware SEO scoring — not just scheduling.

6.2 LinkedIn SEO Engine


6.2.1 Algorithm Architecture (2026)

LinkedIn&apos;s algorithm is unique: it is explicitly designed to prevent virality and instead optimize for professional relevance. It uses a 3-stage pipeline: Quality Filter → Engagement Test → Network+Relevance Ranking.



KEY 2026 UPDATE: LinkedIn introduced the &apos;Depth Score&apos; metric. It now tracks exact dwell time per post.

A user spending 45s reading without liking scores HIGHER than a 2-second like (indicates bot/pity behavior).

Educational content gets 3–5x more reach than other post types (Q1 2026 data, 10,000 post study).

External links in posts get significant reach penalties — use first comment strategy instead.

AI-generated content with generic language is now detectable and de-prioritized.

6.3 X (Twitter) SEO Engine


6.3.1 Algorithm Architecture (2026)

X is the most transparent platform — its algorithm is open-source (GitHub: xai-org/x-algorithm, Jan 2026). The scoring module must implement the 3-stage pipeline: Candidate Sourcing (1,500 posts from 500M) → Grok-Powered Neural Ranking → Heuristics &amp; Filtering.



CRITICAL 2026 CHANGE: Grok AI replaced the legacy recommendation system in Jan 2026. It now reads

every post and video (100M+/day) and ranks by predicted personal relevance — not just engagement counts.

TweepCred Score: Every account has a hidden 0–100 score. Below 65 = only 3 posts considered for distribution.

Premium accounts get +4 to +16 TweepCred boost — the pay-to-play gap is the largest of any platform.

CONFIRMED ENGAGEMENT WEIGHTS (open-source code): Likes × 1, Replies × 13.5, Retweets × 20,

Profile Clicks × 12, Link Clicks × 11, Bookmarks × 10. Author-reply to a reply = +75 (150x a like).

8. Design System &amp; Stitch Design Brief


8.1 Design Direction

Design 04 — Neo-Brutalist — has been selected as the visual direction for RankPulse. This aesthetic communicates: raw intelligence, zero-nonsense precision, and unapologetic authority. The design system uses a bold yellow base (#FFFCE8), thick 2-3px black borders, red (#FF3B00) as the primary action color, and Bebas Neue + Space Grotesk typography. Every UI element is intentional — nothing decorative, everything functional.

This is not a generic SaaS dashboard. RankPulse looks different because it thinks differently. The neo-brutalist direction ensures it stands out in every screenshot, every product hunt launch, and every demo. It is built for creators and agencies who are serious about results.






8.2 Design Concepts — Selected: Design 04 (Neo-Brutalist)



SELECTED: Design 04 — Neo-Brutalist

Aesthetic: Raw grid, thick black borders, yellow (#FFFCE8) base, red (#FF3B00) accent — zero visual noise.

Colors: #FFFCE8 (background), #111111 (primary), #FF3B00 (action/accent), #22C55E (success), #F59E0B (warning)

11. Quota &amp; Cost Optimization Strategy

To build this product efficiently without burning free-tier quotas, the following strategies must be enforced throughout development:




12.1 Development Phase Efficiencies


Use Stitch MCP to scaffold all boilerplate — zero manual CRUD, zero manual auth setup.


Build SEO scoring logic as pure TypeScript functions first — test without any API calls.


Mock all social platform API calls in development using seed data fixtures.


Use Supabase local development (supabase start) — never burn production quota in dev.


12.2 Production Phase Efficiencies


Cache all AI analysis results in Supabase for 24h — identical inputs never re-scored.


Use streaming responses for AI-powered rewrites to reduce TTFB and avoid timeouts.


Batch process scheduled post pre-checks — run all pending analyses in one cron job.


Implement aggressive client-side caching with React Query — staleTime: 30 min for keyword data.


Rate-limit user-facing AI calls by plan tier — free users get 15 analyses/month, pool the rest.


Use Vercel Edge Cache for all public page ISR — landing page cached for 1 hour.


12.3 Stitch MCP Usage Discipline


Always use the most specific skill available — never ask for generic code when a skill exists.


Combine skills in single commands where possible: &apos;scaffold + auth + schema in one pass&apos;.


Reuse generated components via the component library — never regenerate identical UI twice.


Use skills for design system tokens first, then generate all components from the same token set.
